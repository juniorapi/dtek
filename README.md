# ⚡️ DTEK Schedule

Автоматичний моніторинг графіку відключень електроенергії від ДТЕК з редагуванням повідомлень у Telegram.

## ✨ Можливості

- 🤖 Моніторинг **до 3 груп** одночасно через GitHub Actions
- 📱 **Редагує** існуюче повідомлення замість нових — 3 закріплені повідомлення на весь день
- 🚨 **Терміново** оновлює якщо DTEK змінив розклад — не чекає години
- 🔄 **Щогодинне** оновлення часу перевірки навіть якщо розклад не змінився
- 📊 Хронологічний список блоків зі/без світла з тривалістю кожного
- 🕯 Підсумок: скільки годин буде і не буде світла
- 🌙 Тихі сповіщення вночі (00:00–08:00)

## 🚀 Швидкий старт

### 1. Форк репозиторію

Натисніть **Fork** у правому верхньому куті.

### 2. Створення Telegram бота

1. Відкрийте [@BotFather](https://t.me/BotFather) → `/newbot`
2. Збережіть токен у форматі `123456789:ABCdef...`

### 3. Отримання Chat ID

1. Надішліть будь-яке повідомлення боту
2. Відкрийте `https://api.telegram.org/bot<TOKEN>/getUpdates`
3. Знайдіть `"chat":{"id":123456789}` — це ваш Chat ID

### 4. Визначення групи відключень

1. Перейдіть на сайт ДТЕК свого регіону ([КРЕМ](https://www.dtek-krem.com.ua/ua/shutdowns), [ДNEM](https://www.dtek-dnem.com.ua/ua/shutdowns) тощо)
2. Введіть адресу та знайдіть номер групи (наприклад `1.1`)

### 5. Налаштування GitHub Environments

Перейдіть в **Settings → Environments** і створіть environments: `group-1`, `group-2`, `group-3`.

У кожному environment додайте secrets:

| Secret | Опис | Приклад |
|--------|------|---------|
| `TELEGRAM_BOT_TOKEN` | Токен бота від BotFather | `123456789:ABCdef...` |
| `TELEGRAM_CHAT_ID` | ID чату або каналу | `123456789` або `-100123456789` |
| `REGION` | Код регіону ДТЕК | `kr`, `dn`, `k`, `o`, `d` |
| `GROUP` | Номер групи (без GPV) | `1.1` |
| `GROUP_LABEL` | Підпис адреси у повідомленні | `м. Київ, вул. Хрещатик 1` |

У **Settings → Secrets → Actions** (репозиторій) додайте:

| Secret | Опис |
|--------|------|
| `PAT` | Personal Access Token для збереження артефактів |

#### Коди регіонів

| Код | Регіон |
|-----|--------|
| `k` | Київ |
| `kr` | Київська обл. |
| `dn` | Дніпропетровська обл. |
| `o` | Одеська обл. |
| `d` | Донецька обл. |

#### Створення PAT

1. **Settings → Developer settings → Personal access tokens → Fine-grained tokens**
2. **Generate new token** → назва: `dtek-schedule`
3. **Repository access** → обрати цей репозиторій
4. Permissions: `Read access to metadata` + `Read and Write access to code`
5. Скопіюйте токен

### 6. Додати підпис адреси групи (опційно)

У кожному environment додайте secret `GROUP_LABEL` зі своєю адресою, наприклад:

```
м. Київ, вул. Хрещатик 1
```

Якщо `GROUP_LABEL` не заданий — буде показано назву регіону та номер групи.

### 7. Запуск

1. Перейдіть на вкладку **Actions**
2. Оберіть **"Run schedule for all groups"**
3. Натисніть **Run workflow**

Після цього workflow запускається автоматично кожні 5 хвилин і редагує повідомлення раз на годину або одразу при зміні розкладу DTEK.

## 🛠 Локальний запуск

### Вимоги

- Node.js ≥ 22.15.1 (версія в `.nvmrc`)

### Встановлення

```bash
git clone https://github.com/YOUR_USERNAME/dtek-schedule.git
cd dtek-schedule
npm install
npx playwright install chromium
```

### Налаштування `.env`

```env
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
TELEGRAM_CHAT_ID=-100123456789
REGION=kr
GROUP=1.1
```

### Запуск

```bash
npm start
```

> **Примітка:** Локально сайт DTEK може блокуватись Incapsula (антибот захист). У GitHub Actions працює коректно.

## 🔍 Як це працює

1. **Отримання даних** — Playwright відкриває сайт ДТЕК та парсить JSON з `fact = {...}`
2. **Генерація розкладу** — формуються блоки по 30/60 хв зі станом ON/OFF
3. **Розумне оновлення:**
   - DTEK змінив розклад → **терміново** редагує повідомлення
   - Минула година → **планово** редагує (оновлює час перевірки)
   - Інакше → пропускає, логує час до наступного оновлення
4. **Збереження стану** — `artifacts/last-message-{group}.json` комітиться в репо між запусками

## ⚙️ Структура артефактів

```json
{
  "message_id": 1234,
  "date": 1740000000,
  "text": "...",
  "apiUpdate": "01.03.2026 10:00",
  "lastEditedAt": 1740003600000
}
```

## 🕐 Стани електропостачання

| API значення | Стан |
|-------------|------|
| `yes` | Вся година зі світлом |
| `no` | Вся година без світла |
| `first` | 00–30 без світла, 30–60 зі світлом |
| `second` | 00–30 зі світлом, 30–60 без світла |

## ⚠️ Обмеження

- Не працює якщо DTEK змінить структуру сторінки
- Telegram не сповіщає звуком про **редагування** повідомлень (тільки нові дають сповіщення)
- Максимум 5 повторних спроб при помилці з затримкою 5 сек
